## Technical Implementation

### Data Structure

Updated the `PaymentApp` interface to include an optional `logoUrl` field:

```typescript
export interface PaymentApp {
  name: string;
  icon: string; // URL or emoji (fallback)
  logoUrl?: string; // Logo image URL
  deepLink: string;
  webUrl: string;
  androidPackage?: string;
  iosScheme?: string;
  description: string;
}
```

### Component Updates

**QuickLinks Component** (`/src/components/dashboard/QuickLinks.tsx`):

```tsx
<div className="flex items-center gap-2 w-full mb-2">
  {app.logoUrl ? (
    <img 
      src={app.logoUrl} 
      alt={`${app.name} logo`}
      className="h-8 w-8 object-contain rounded"
    />
  ) : (
    <span className="text-2xl">{app.icon}</span>
  )}
  <span className="font-semibold text-sm">{app.name}</span>
  <ExternalLink className="h-3 w-3 ml-auto opacity-50" />
</div>
```

### Fallback Mechanism

- If `logoUrl` is provided, display the logo image
- If `logoUrl` is not available, fall back to emoji icon
- Ensures backward compatibility and graceful degradation

## Maintenance

### Adding New Logos

1. **Obtain Logo Image**:
   - Search for official brand logo
   - Ensure proper licensing/usage rights
   - Prefer square or icon versions

2. **Upload to CDN**:
   - Use image search tool or upload to CDN
   - Get the full URL

3. **Update Configuration**:
   ```typescript
   {
     name: 'App Name',
     icon: 'üîµ', // Fallback emoji
     logoUrl: 'https://miaoda-site-img.s3cdn.medo.dev/images/[uuid].jpg',
     // ... other fields
   }
   ```

4. **Test**:
   - Verify logo displays correctly
   - Check on mobile and desktop
   - Test in light and dark modes

### Updating Existing Logos

1. Upload new logo to CDN
2. Update the `logoUrl` in `/src/config/paymentApps.ts`
3. Test the change
4. Clear browser cache if needed
